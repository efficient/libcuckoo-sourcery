template<class Key, class T, class Hash = std::hash<Key>, class Pred = std::equal_to<Key>>
cuckoohash_map< Key, T, Hash, Pred >::cuckoohash_map ( size_t  hashpower_init = HASHPOWER_DEFAULT)
inlineexplicit

The constructor creates a new hash table with a hashpower of hashpower_init. The number of buckets in the new hash table will be 2hashpower_init. Since each bucket can hold SLOT_PER_BUCKET items, the table will have 2hashpower_init × SLOT_PER_BUCKET slots to store items in. If the constructor fails, it will throw an exception.

template<class Key, class T, class Hash = std::hash<Key>, class Pred = std::equal_to<Key>>
bool cuckoohash_map< Key, T, Hash, Pred >::rehash ( size_t  n)
inline

rehash will size the table using a hashpower of n. Note that the number of buckets in the table will be 2n after expansion, so the table has 2n × SLOT_PER_BUCKET slots to store items in. If n is not larger than the current hashpower, then the function does nothing. It returns true if the table expansion succeeded, and false otherwise. rehash can throw an exception if the expansion fails to allocate enough memory for the larger table.

Definition at line 393 of file cuckoohash_map.hh.

template<class Key, class T, class Hash = std::hash<Key>, class Pred = std::equal_to<Key>>
bool cuckoohash_map< Key, T, Hash, Pred >::reserve ( size_t  n)
inline

reserve will size the table to have enough slots for at least n elements. If the table can already hold that many elements, the function has no effect. Otherwise, the function will expand the table to a hashpower sufficient to hold n elements. It will return true if there was an expansion, and false otherwise. reserve can throw an exception if the expansion fails to allocate enough memory for the larger table.
